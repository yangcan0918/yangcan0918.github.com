hexo博客系统

Posted on 2014-11-01

一直想找一个满意的博客系统，曾经看过很多人的博客，发现很多都是使用51cto、CSDN、wordpress,oschina、新浪等等很多的博客系统，但是大部分都是不支持macdown格式的，写起来不美观，同时也比较麻烦。最近一直在观察别人使用的博客系统(一般是在看别人的技术文章时发现的),前段时间，发现了几款博客系统，也尝试用了几款，如Jekkyll、scriptorg、wordpress等等博客系统，最后发现还是存在各种的问题，所以继续观察是否有别的更好一点的博客系统，最后终于在看技术文档的时候，发现有一部分使用hexo写，感觉还不错，所以就大胆的尝试了一下，发现还是值得一试，发现原来还是有很多免费的资源可以使用的。下面我就来总结一下hexo博客系统的搭建。

hexo介绍

hexo来历

hexo出自台湾大学生tommy351之手，是一个基于Node.js的静态博客程序，其编译上百篇文字只需要几秒。hexo生成的静态网页可以直接放到GitHub Pages，BAE，SAE等平台上。

hexo博客系统的使用过程

在自己的电脑上安装hexo系统，具体安装见安装部分，可以在windows,linux,mac系统上进行安装和部署。安装完后，直接使用hexo init blog即可创建以blog命名的blog目录，里面存放博客系统的很多文件，进入博客目录，使用hexo new “name” 即可创建一篇文章，文章在blog/source/_posts目录下，添加文章内容，使用hexo generate命令即可生成静态站点，使用hexo server即可启动本地server.这时一个本地的博客系统就建成了。在这里只是简单的说一下，具体的流程，在后续还会有更多的内容。如，修改博客主题，将博客放到github上，域名绑定，百度分享、微博秀，多说评论，添加小图标，添加分类，标签云，友情链接，百度统计，文章摘要，导航栏添加等等内容

hexo的安装和初始化

hexo可以安装在windows、linux、mac上，我这里是使用mac进行部署

安装依赖包

安装git

brew install git

安装node.js

brew install node

安装hexo

npm install -g hexo

开始尝试使用hexo

创建Blog

hexo init blog # 创建了一个以blog命名的博客目录

进入目录

cd blog #操作都是在博客目录进行

创建新文章

hexo new "New Post" #文章默认都是存放在blog/source/_posts，还可以创建其他类型的
使用格式是hexo new [layout] <title>，还有layout还有photo，page等等

编写文章

title: postname #文章页面上的显示名称，可以任意修改，不会出现在URL中
date: 2014-10-31 15:30:16 #文章生成时间，一般不改，当然也可以任意修改
categories: #文章分类目录，可以为空，注意:后面有个空格
tags: #文章标签，可空，多标签请用格式[tag1,tag2,tag3]，注意:后面有个空格
---
这里开始使用markdown格式输入你的正文。

产生静态站点

hexo generate

启动本地server

hexo server

在本机访问

localhost:4000 #默认开启的是4000端口，只能本机访问。具体修改可以在_config.yml中进行，并且可以在该文件中配置github账号，使用hexo deploy发布到github上。

hexo目录详解

  • .deploy：执行hexo deploy命令部署到GitHub上的内容目录
  • public：执行hexo generate命令，输出的静态网页内容目录
  • scaffolds：layout模板文件目录，其中的md文件可以添加编辑
  • scripts：扩展脚本目录，这里可以自定义一些javascript脚本
  • source：文章源码目录，该目录下的markdown和html文件均会被hexo处理。该页面对应repo的根目录，404文件、favicon.ico文件，CNAME文件等都应该放这里，该目录下可新建页面目录。
    • _drafts：草稿文章
    • _posts：发布文章
  • themes：主题文件目录
  • _config.yml：全局配置文件，大多数的设置都在这里
  • package.json：应用程序数据，指明hexo的版本等信息，类似于一般软件中的关于按钮

添加友情链接widget

<div class="widget tag">

主页文章显示摘要

编辑md文件的时候，在要作为摘要的文字后面添加<!--more-->即可。

设置百度统计、Google Analytics

到百度统计和Google Analytics（可能需要穿墙）注册账号、添加网站地址即可，很简单，不详述了。需要注意的是，设置好之后数据要过几个小时才能出现，不用着急。

文章中插入图片

使用markdown写文章，插入图片的格式为![图片名称](链接地址)，这里要说的是链接地址怎么写。对于hexo，有两种方式：

  • 使用本地路径：在hexo/source目录下新建一个img文件夹，将图片放入该文件夹下，插入图片时链接即为/img/图片名称。
  • 使用微博图床，地址http://weibotuchuang.sinaapp.com/，将图片拖入区域中，会生成图片的URL，这就是链接地址。

加入fork me on github
